## Cross-Phase Integration

**Integration is verified at the highest level** via `cluster/21_submit_pipeline.sh` (master orchestrator), which:

1. Runs a local pre-flight pytest gate on `test_numpyro_models_2cov.py` before any sbatch call (saves ~150 GPU-hours on broken L2 hook)
2. Chains 9 pipeline steps via `sbatch --parsable` + `--dependency=afterok` exclusively (zero `afterany` — verified in STATE.md Phase 21 decisions)
3. Step 21.6 uses a proper SLURM wrapper (`cluster/21_6_dispatch_l2.slurm`, `--time=14:00:00`) around the canonical sbatch-wait dispatcher to survive the 12h M6b subscale worst case
4. Unified exit-0 semantics (PROCEED_TO_AVERAGING and NULL_RESULT both exit 0) enable afterok to advance on either valid scientific outcome; exit 1 is reserved for genuine errors

**Wire-up confirmed between phases:**

- Phase 13 infrastructure (`numpyro_helpers`, `bayesian_diagnostics`, `bayesian_summary_writer`) consumed by all downstream phases (16, 17, 18, 21)
- Phase 16 level2_design.py consumed by Phase 21 winner L2 refit (via 21-07 `_fit_two_covariate_l2` and `--subscale` flag)
