   ```python
   checkpoint_path = Path(args.checkpoint_dir) / "m4_warmup_state.pkl"
   rng_key = jax.random.PRNGKey(args.seed)
   model_kwargs = {"participant_data_stacked": participant_data_stacked}

   start_time = time.time()

   if checkpoint_path.exists():
       print(f"\n[M4H-04] Resuming from warmup checkpoint: {checkpoint_path}")
       with open(checkpoint_path, "rb") as f:
           warmup_state = pickle.load(f)
       mcmc.post_warmup_state = warmup_state
       mcmc.run(warmup_state.rng_key, **model_kwargs)
   else:
       print("\n[M4H-04] Running warmup phase...")
       mcmc.warmup(rng_key, **model_kwargs)
       # Save warmup state with jax.device_get to avoid GPU array serialization
       print(f"[M4H-04] Saving warmup state to {checkpoint_path}...")
       checkpoint_path.parent.mkdir(parents=True, exist_ok=True)
       warmup_state_cpu = jax.device_get(mcmc.post_warmup_state)
       with open(checkpoint_path, "wb") as f:
           pickle.dump(warmup_state_cpu, f)
       print("[M4H-04] Warmup state saved. Running sampling phase...")
       mcmc.run(mcmc.post_warmup_state.rng_key, **model_kwargs)

   wall_time = time.time() - start_time
   print(f"\n[TIMING] Total MCMC wall time: {wall_time/3600:.2f} hours")
   ```

   ```python
   from config import MODEL_REGISTRY, EXPECTED_PARAMETERIZATION
   
   model_name = "wmrl_m4"
   model_params = MODEL_REGISTRY[model_name]["params"]
   
   # Build InferenceData for convergence check
   # M4-specific pointwise log-lik (inline, since bayesian_diagnostics doesn't support M4)

   pointwise_loglik = compute_m4_pointwise_loglik(mcmc, participant_data_stacked)

   idata = build_inference_data_with_loglik(

   )
   
   # Convergence check
   convergence_summary = az.summary(idata, var_names=model_params)
   max_rhat = float(convergence_summary["r_hat"].max())
   min_ess = float(convergence_summary["ess_bulk"].min())
   extra = mcmc.get_extra_fields()
   n_div = int(extra["diverging"].sum()) if "diverging" in extra else 0
   converged = max_rhat < 1.01 and min_ess > 400 and n_div == 0
   
   if not converged:
       print(f"\n[CONVERGENCE GATE FAILED] max_rhat={max_rhat:.4f}, "
   ```

   ```python
   print("\n>> Pareto-k gating (M4H-05)...")
   loo_result = az.loo(idata, pointwise=True)
   pareto_k = loo_result.pareto_k.values
   frac_bad = float((pareto_k > 0.7).mean())
   
   pareto_report = {
       "pareto_k_frac_bad": frac_bad,
       "pareto_k_threshold": 0.05,
       "n_trials_checked": int(pareto_k.size),
       "n_trials_bad": int((pareto_k > 0.7).sum()),
   }
   
   if frac_bad > 0.05:
       print(f"[M4H-05 FALLBACK] {100*frac_bad:.1f}% trials have Pareto-k > 0.7")
       print("LOO is unreliable for M4. Use MLE AIC track for cross-model comparison.")
       print("LOO reported as standalone M4 quality metric only (not cross-model).")
       pareto_report["loo_unreliable"] = True
       pareto_report["fallback"] = "mle_aic_track"
       pareto_report["loo_elpd"] = float(loo_result.loo)
   else:
       print(f"[M4H-05 PASS] {100*frac_bad:.1f}% Pareto-k > 0.7 (threshold: 5%)")
       pareto_report["loo_unreliable"] = False
       pareto_report["loo_elpd"] = float(loo_result.loo)
   
   # Write Pareto-k report
   report_path = bayesian_dir / "wmrl_m4_pareto_k_report.md"

   with open(json_path, "w") as f:
       json.dump(pareto_report, f, indent=2)
   ```

**Helper function: `compute_m4_pointwise_loglik`**

This is the M4-specific replacement for `compute_pointwise_log_lik` from bayesian_diagnostics.py. It must:
- Extract posterior samples from `mcmc.get_samples()`
- For each (chain, sample) pair, for each participant, call `wmrl_m4_multiblock_likelihood_stacked` with `return_pointwise=True` (if available) or compute per-block log-liks
- Return array of shape `(chains, samples, n_participants, n_blocks * MAX_TRIALS_PER_BLOCK)`

IMPORTANT: Check if `wmrl_m4_multiblock_likelihood_stacked` supports `return_pointwise=True`. If NOT, use a simpler approach: compute total log-lik per participant as a single scalar (shape `(chains, samples, n_participants)`) and skip the trial-level Pareto-k (use participant-level instead). This is acceptable because Pareto-k > 0.7 is near-certain for LBA anyway.

Practical approach: Use `jax.vmap` over the samples dimension for each participant, calling the likelihood with individual-level parameter draws. The result per participant is a scalar log-lik. Stack into shape `(n_chains, n_samples, n_participants)` and use this for `build_inference_data_with_loglik`. The filter_padding_from_loglik step may need to be adapted (pass through if shape doesn't match expectations, since M4 log-lik may be participant-level rather than trial-level).
