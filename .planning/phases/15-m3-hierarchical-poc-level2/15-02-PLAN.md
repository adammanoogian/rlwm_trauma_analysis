**1. `compute_shrinkage_report(idata, param_names)`**

Add after the existing `build_inference_data_with_loglik` function.

```python
def compute_shrinkage_report(
    idata: "az.InferenceData",
    param_names: list[str],
) -> dict[str, float]:
```

Implementation:
- For each param in param_names:
  - Get posterior draws: `idata.posterior[param].values` -- shape `(chains, draws, n_participants)`.
  - Reshape to `(total_draws, n_participants)`.
  - `var_indiv = np.var(flat)` -- total variance across ALL draws AND participants.
  - `var_group_mean = np.var(flat.mean(axis=1))` -- variance of per-draw group mean.
  - `shrinkage = 1.0 - var_indiv / (var_group_mean + 1e-10)`.
  - Store in results dict.
- Return `dict[str, float]` mapping param name to shrinkage value.
- Import numpy (not jax.numpy) for this function -- it operates on ArviZ data.
- NumPy-style docstring.

**2. `write_shrinkage_report(shrinkage: dict, output_path: Path, threshold: float = 0.3)`**

Writes a markdown report to the given path.

```python
def write_shrinkage_report(
    shrinkage: dict[str, float],
    output_path: Path,
    *,
    threshold: float = 0.3,
) -> Path:
```

Implementation:
- Write header: `# Shrinkage Diagnostic Report`
- Write formula: `Shrinkage = 1 - var(individual draws) / var(per-draw group mean)`
- Write table: `| Parameter | Shrinkage | Status |`
- Status: "identified" if >= threshold, "WARNING: poorly identified" if < threshold.
- Write summary: count of identified vs poorly-identified parameters.
- Write note: "Parameters with shrinkage < 0.3 should be treated as descriptive only for downstream inference."
- Return the output path.

**3. `filter_padding_from_loglik(pointwise_loglik, participant_data_stacked)`**

Helper that removes padding positions before WAIC/LOO.

```python
def filter_padding_from_loglik(
    pointwise_loglik: jnp.ndarray,
    participant_data_stacked: dict,
) -> jnp.ndarray:
```

Implementation:
- `pointwise_loglik` has shape `(chains, samples, n_participants, n_blocks * max_trials)`.
- For each participant, build a combined mask from their `masks_stacked` (flatten n_blocks * max_trials).
- Use NaN sentinel approach:
  - For each participant index, get their masks_stacked (n_blocks, max_trials).
  - Flatten to (n_blocks * max_trials,).
  - Where mask == 0, set pointwise_loglik[..., participant_idx, trial_idx] = NaN.
  - Return the modified array. The caller passes this to `build_inference_data_with_loglik`.
  - ArviZ 0.23.4 drops NaN observations in az.waic() and az.loo().
