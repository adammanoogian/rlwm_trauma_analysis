**Architecture for each pscan block likelihood:**

The pscan variant replaces the inner `lax.scan` with a two-phase approach:
1. **Phase 1 (parallel):** Pre-compute Q and WM trajectories for the entire block using `associative_scan_q_update` and `associative_scan_wm_update` (for WM-RL models).
2. **Phase 2 (sequential):** Run a `lax.scan` over trials for the policy computation only. This scan reads pre-computed Q[t] and WM[t] arrays instead of carrying Q/WM state. It still carries `last_action` (for perseveration models) and accumulates log-likelihood.

**Model-specific details:**

**M1: Q-Learning (`q_learning_block_likelihood_pscan`, `q_learning_multiblock_likelihood_stacked_pscan`)**
- Phase 1: `associative_scan_q_update` only (no WM)
- Phase 2: Sequential scan reads `Q_for_policy[t, stimulus_t]`, computes softmax, epsilon noise, log_prob. No perseveration carry.
- Signature matches `q_learning_block_likelihood` exactly (same params, same return type, same `return_pointwise` support)

**M2: WM-RL (`wmrl_block_likelihood_pscan`, `wmrl_multiblock_likelihood_stacked_pscan`)**
- Phase 1: `associative_scan_q_update` + `associative_scan_wm_update` (decay-only pass for policy)
- Phase 2: Sequential scan reads Q[t] and WM_decayed[t], computes omega, hybrid policy, epsilon noise. No perseveration carry.
- `wm_for_policy[t]` used for softmax(WM[stimulus_t])

**M3: WM-RL+kappa (`wmrl_m3_block_likelihood_pscan`, `wmrl_m3_multiblock_likelihood_stacked_pscan`)**
- Phase 1: Same as M2 (Q + WM scans)
- Phase 2: Sequential scan reads Q[t], WM_decayed[t], carries `last_action` for perseveration kernel. Applies `(1-kappa)*P_noisy + kappa*Ck`.
- The `last_action` carry makes this hybrid: Q/WM are O(log T) parallel, but the policy loop is O(T) sequential.

**M5: WM-RL+phi_rl (`wmrl_m5_block_likelihood_pscan`, `wmrl_m5_multiblock_likelihood_stacked_pscan`)**
- Phase 1: Q-update uses `associative_scan_q_update` but with an additional Q-forgetting step. The forgetting recurrence is `Q_t = (1-phi_rl)*Q_{t-1} + phi_rl*q_init` applied BEFORE the learning update. This means the combined single-trial operator is: first decay (`a=1-phi_rl, b=phi_rl*q_init`), then update (`a=1-alpha, b=alpha*r`). The composition of two affine operators is `(a2*a1, a2*b1+b2)`. Compute the composed `a_t, b_t` per trial and feed to `affine_scan`.
  - If the trial is active and valid: `a_t = (1-alpha_t)*(1-phi_rl)`, `b_t = (1-alpha_t)*phi_rl*q_init + alpha_t*reward_t`
  - If inactive or padding: `a_t = (1-phi_rl)`, `b_t = phi_rl*q_init` (decay only, no learning)
  - Note: for M5, even inactive trials see Q decay (phi_rl applies globally). Check the sequential M5 code to confirm this.
- Phase 1 also: WM update same as M2/M3
- Phase 2: Sequential scan with `last_action` carry (perseveration via kappa)

**M6a: WM-RL+kappa_s (`wmrl_m6a_block_likelihood_pscan`, `wmrl_m6a_multiblock_likelihood_stacked_pscan`)**
- Phase 1: Same as M3 (Q + WM scans, no phi_rl)
- Phase 2: Sequential scan carries `last_actions` (per-stimulus, shape `(num_stimuli,)`) instead of scalar `last_action`. The perseveration kernel `Ck` is stimulus-specific: `one_hot(last_actions[stimulus_t])`.

**M6b: WM-RL+dual (`wmrl_m6b_block_likelihood_pscan`, `wmrl_m6b_multiblock_likelihood_stacked_pscan`)**
- Phase 1: Same as M3 (Q + WM scans)
- Phase 2: Sequential scan carries both `last_action` (global) and `last_actions` (per-stimulus). Dual perseveration: `kappa = kappa_total * kappa_share` (global), `kappa_s = kappa_total * (1 - kappa_share)` (stimulus-specific). Combined kernel: `P = (1-kappa-kappa_s)*P_noisy + kappa*Ck_global + kappa_s*Ck_stimulus`.

**Multiblock wrappers:**
Each `*_multiblock_likelihood_stacked_pscan` mirrors the existing `*_multiblock_likelihood_stacked` structure: uses `lax.fori_loop` over blocks, calling `*_block_likelihood_pscan` inside. Signature matches the sequential version exactly (same params, same return). Support `return_pointwise` if the sequential version does.

**CRITICAL implementation details:**
- Read the sequential M5 `wmrl_m5_block_likelihood` carefully to understand whether Q-forgetting (phi_rl) applies at every trial or only at active trials. The pscan must match this behavior exactly.
- For M6a/M6b, read the sequential code to understand the exact perseveration mixing formula and last_actions update pattern.
- The `mask` (valid) array handling must be identical: padding trials get `log_prob * 0.0`, Q/WM identity at padding positions.
- Block-level Q and WM initialization: both reset to `q_init` and `wm_init` at each block start (no carry across blocks).
