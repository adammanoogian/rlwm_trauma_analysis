1. **AR(1) Linear Recurrence Formulation** -- Full worked example showing how `Q_t(s,a) = (1-alpha)*Q_{t-1}(s,a) + alpha*r_t` maps to `x_t = a_t*x_{t-1} + b_t` with the associative operator `(a2,b2) o (a1,b1) = (a2*a1, a2*b1+b2)`. Include proof that the observed-data log-likelihood evaluation reduces to computing Q-values via this recurrence.

2. **WM Decay and Overwrite** -- Show how `WM_t = (1-phi)*WM_{t-1} + phi*WM_0` is the same affine form, and how the hard overwrite `WM(s,a) <- r` is encoded as `a_t=0, b_t=r` (multiplicative reset). Derive formally that all information before a reset position is zeroed out.

3. **Which RLWM Components Are Linear vs Non-Linear** -- Explicit table:
   - Linear (parallelizable): Q-value update, WM decay, WM overwrite
   - Non-linear (sequential): WM-Q mixing weight, softmax policy, epsilon noise, perseveration kernel
   - Why: the non-linear parts depend on the Q AND WM values simultaneously, or depend on the action taken (which is a non-linear function of the policy)

4. **Related Work** (one paragraph + citation each):
   - PaMoRL (NeurIPS 2024) -- PETE algorithm for TD-lambda eligibility traces
   - S4/Mamba -- data-dependent decay in selective SSMs, same associative scan
   - Parallel Kalman smoothers (Sarkka & Garcia-Fernandez 2021) -- affine operator in linear Gaussian state space
   - Brief note on DEER (ICLR 2024) for non-linear components (Phase 20)

5. **Alpha Approximation** -- Document that `alpha_t = alpha_pos if reward==1 else alpha_neg` is an approximation of `alpha_t = alpha_pos if delta>0 else alpha_neg`. Exact when Q(s,a) < 1 for rewarded trials and Q(s,a) > 0 for unrewarded. Expected agreement < 1e-5 for typical parameters.

6. **Numerical Stability** -- Edge cases for alpha near 0/1, phi near 0/1. Float32 underflow at `(1-alpha)^T` for alpha > 0.9 and T=100. Tolerance thresholds.

**In `jax_likelihoods.py`**, add three new functions AFTER the existing helper functions (after `apply_epsilon_noise`, around line 280) but BEFORE any model-specific likelihood functions. Keep all existing sequential functions untouched.

1. **`affine_scan(a_seq, b_seq, x0)`** -- Generic parallel prefix scan for `x_t = a_t * x_{t-1} + b_t`.
   - `a_seq`: shape `(T, ...)`, multiplicative coefficients
   - `b_seq`: shape `(T, ...)`, additive coefficients
   - `x0`: initial value, broadcastable to `(...)`
   - Returns: `x_all` shape `(T, ...)` where `x_all[t]` is the state AFTER applying operator at position t
   - Implementation: Prepend `(1.0, x0)` as the identity+init element, call `jax.lax.associative_scan` with the affine operator `(a_r*a_l, a_r*b_l + b_r)`, return `x_all[1:]` (drop the prepended init)
   - The operator must work element-wise on arbitrary trailing dimensions (JAX broadcasts naturally)

2. **`associative_scan_q_update(stimuli, actions, rewards, masks, alpha_pos, alpha_neg, q_init, num_stimuli, num_actions)`** -- Compute all T Q-value tables via parallel scan.
   - Returns: `Q_for_policy` shape `(T, num_stimuli, num_actions)` where `Q_for_policy[t]` is the Q-table BEFORE trial t's update (for policy computation at trial t)
   - Build `(T, S, A)` sequences of `a_t` and `b_t`:
     - `sa_mask = one_hot(stimuli, S)[:,:,None] * one_hot(actions, A)[:,None,:]` shape `(T, S, A)`
     - `active = sa_mask * masks[:, None, None]` 
     - Alpha selection: `alpha_t = where(rewards[:,None,None] == 1.0, alpha_pos, alpha_neg)` (data-dependent approximation)
     - `a_seq = where(active, 1.0 - alpha_t, 1.0)` (identity at inactive positions)
     - `b_seq = where(active, alpha_t * rewards[:,None,None], 0.0)`
   - Call `affine_scan(a_seq, b_seq, x0=q_init * ones((S,A)))`
   - The result `Q_all[t]` = Q AFTER update at trial t. For policy at trial t, we need Q BEFORE update t. So prepend `Q_init` and drop the last: `Q_for_policy = concat([Q_init[None], Q_all[:-1]], axis=0)`

3. **`associative_scan_wm_update(stimuli, actions, rewards, masks, phi, wm_init, num_stimuli, num_actions)`** -- Compute WM trajectories.
   - **BEFORE implementing:** Read the existing `wmrl_m3_block_likelihood` sequential WM update loop in `jax_likelihoods.py` and confirm the decay formula is `WM_decayed = (1 - phi) * WM_table + phi * WM_baseline`. This maps to affine coefficients `a_t = (1-phi), b_t = phi*wm_init`. Verify these match the coefficients used below before proceeding.
   - Returns tuple: `(wm_for_policy, wm_after_update)` both shape `(T, S, A)`
   - `wm_for_policy[t]` = WM after decay but before overwrite at trial t (used for policy)
   - `wm_after_update[t]` = WM after both decay and overwrite at trial t
   - **Pass 1 (decay only):** `a_seq = full((T,S,A), 1.0-phi)`, `b_seq = full((T,S,A), phi*wm_init)`. Call `affine_scan`. Result gives `wm_for_policy` (prepend init, drop last, same as Q indexing).
   - **Pass 2 (decay + overwrite):** At overwrite positions (active and valid): `a=0.0, b=reward`. At non-active: `a=1.0-phi, b=phi*wm_init`. At padding: `a=1.0, b=0.0`. Call `affine_scan`. Result gives `wm_after_update` (prepend init, drop last).

2. **`test_affine_scan_identity`** -- All `a_t=1, b_t=0`: output should be `x0` everywhere.

For the sequential reference Q-values, implement a simple Python/JAX sequential loop in the test file (not using the full block_likelihood, which has too many coupled components). The loop should be:
```python
Q = q_init * ones((S, A))
Q_history = []
for t in range(T):
    Q_history.append(Q.copy())
    if mask[t]:
        s, a, r = stimuli[t], actions[t], rewards[t]
        q_old = Q[s, a]
        delta = r - q_old
        alpha = alpha_pos if delta > 0 else alpha_neg  # exact delta-sign rule
        Q = Q.at[s, a].set(q_old + alpha * delta)
```
This gives the ground truth. The pscan uses the reward-based alpha approximation, so we expect very close but not exact agreement.
