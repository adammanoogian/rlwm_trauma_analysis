<objective>
Build step 21.4 — convergence + fit-quality audit over the six baseline posteriors.

Purpose: Enforce the Baribault & Collins (2023, DOI 10.1037/met0000554) convergence gate as a hard pipeline block between 21.3 and 21.5. Any model failing R-hat ≤ 1.05 / ESS_bulk ≥ 400 / 0 divergences / BFMI ≥ 0.2 after auto-bump is excluded from LOO+stacking. If < 2 models pass, the pipeline must fail loud (ROADMAP Phase 21 success criterion #2). This resolves the ambiguity "should we let a non-converged model participate in stacking" — answer: no.

Output: `output/bayesian/21_baseline/convergence_report.md` with per-model PASS/FAIL verdict and pipeline action flag. Plus CSV `convergence_table.csv` for machine parsing.
</objective>

Create `scripts/21_baseline_audit.py`. Header with module docstring citing Baribault & Collins (2023).

**CLI:** `--baseline-dir output/bayesian/21_baseline/ --output-dir output/bayesian/21_baseline/ --rhat-threshold 1.05 --ess-threshold 400 --bfmi-threshold 0.2`.

**Workflow:**

1. Define the expected model list: `MODELS = ["qlearning", "wmrl", "wmrl_m3", "wmrl_m5", "wmrl_m6a", "wmrl_m6b"]`.

2. For each model:
   a. Attempt to load `{baseline_dir}/{model}_posterior.nc` via `arviz.from_netcdf`. If file missing: record status="missing", gate=FAIL, action="EXCLUDED_MISSING_FILE".
   b. Load the MODEL_REGISTRY[model]["params"] list and run `az.summary(idata, var_names=params)` — compute max R-hat, min ESS_bulk.
   c. Read divergence count from `idata.sample_stats.diverging.values.sum()` if available (fallback to 0 if not in InferenceData).
   d. Compute BFMI via `az.bfmi(idata)` — returns per-chain values. Use `min(bfmi_per_chain)`.
   e. Apply the gate: `converged = (max_rhat <= 1.05) AND (min_ess >= 400) AND (div == 0) AND (bfmi_min >= 0.2)`.
   f. Run a lightweight PPC summary (do NOT re-simulate; just read `{model}_ppc_results.csv` from `{baseline_dir}/{model}_ppc_results.csv` — the plan-21-04 Task 1 PPC-path fix guarantees this file is written to the same 21_baseline/ subdir as the NetCDF). Record `blocks_covered / total_blocks`. **If missing (despite the plan 21-04 patch):** set `ppc_coverage = "WARNING_FILE_MISSING"` and add the model to the WARNINGS section of `convergence_report.md` — do NOT silently report "not_available". A missing PPC CSV after the Task-1 patch indicates an upstream bug and must be surfaced loudly.

3. Assemble table with columns: `model, max_rhat, min_ess_bulk, n_divergences, min_bfmi, ppc_coverage, gate_status, pipeline_action`. Pipeline actions: `PROCEED_TO_LOO` (gate PASSES), `EXCLUDED_DIVERGENCES`, `EXCLUDED_RHAT`, `EXCLUDED_ESS`, `EXCLUDED_BFMI`, `EXCLUDED_MISSING_FILE`.

4. Write:
   - `{output_dir}/convergence_table.csv`
   - `{output_dir}/convergence_report.md` — human-readable verdict: for each model a section with metrics + pipeline action; a "Summary" header at top listing `n_passing`, `n_excluded`, `models_proceeding_to_loo`.

5. Exit code logic:
   - `n_passing >= 2` → exit 0 (pipeline continues).
   - `n_passing < 2` → exit 1 (pipeline blocks — insufficient models for stacking).
   - Log a loud warning `[PIPELINE BLOCK] Only <n> models passed convergence gate; step 21.5 requires ≥ 2` on exit 1.

6. If any of the 6 models is absent, log a warning but do not fail immediately — allow the pipeline to proceed if ≥ 2 passes.

Follow CLAUDE.md: `from __future__ import annotations`, Python 3.10+ types, absolute imports, NumPy docstrings, line length 88.
