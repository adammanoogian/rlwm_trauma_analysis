<objective>
Build step 21.5 — PSIS-LOO + stacking weights (primary) and RFX-BMS + PXP (secondary) over convergence-gate-passing baseline models.

Purpose: Principled model ranking without circular MLE pre-selection. Primary comparison is LOO stacking per Yao, Vehtari, Simpson, Gelman (2018, *Bayesian Analysis* 13(3):917–1007, DOI 10.1214/17-BA1091). Secondary is RFX-BMS with protected exceedance probability per Stephan et al. (2009) and Rigoux et al. (2014). Hard Pareto-k < 0.7 gate on > 99% of observations per model per Vehtari, Gelman, Gabry (2017, *Stat Comput* 27:1413–1432, DOI 10.1007/s11222-016-9696-4).

**This is a blocking-checkpoint plan (`autonomous: false`).** The winner selection has scientific consequences (drives which models get L2-refit in step 21.6), so the script writes a winner_report.md and *pauses* so the user confirms before the pipeline orchestrator advances.

Output: three files under `output/bayesian/21_baseline/` — the LOO/stacking table, the RFX-BMS/PXP table, and the winner_report.md. Exit codes encode (0, advance), (1, abort), (2, ambiguous — user must decide winner set via CLI rerun with `--force-winners`).
</objective>

  <action>
Create `scripts/21_compute_loo_stacking.py`. Header docstring: cite Yao et al. (2018), Stephan et al. (2009), Rigoux et al. (2014), Vehtari et al. (2017) with DOIs.

**CLI:** `--baseline-dir output/bayesian/21_baseline/`, `--output-dir output/bayesian/21_baseline/`, `--pareto-k-threshold 0.7`, `--pareto-k-pct-threshold 1.0`, `--stacking-winner-threshold 0.5`, `--combined-winner-threshold 0.8`, `--weak-winner-threshold 0.10`, `--force-winners` (comma-separated override, used to manually resume after the checkpoint).

**Workflow:**

1. Parse `{baseline_dir}/convergence_table.csv` (produced by plan 21-05). Select rows with `pipeline_action == "PROCEED_TO_LOO"`. If < 2 rows, exit 1 with message "[ABORT] fewer than 2 models passed convergence".

2. For each eligible model, load `{baseline_dir}/{model}_posterior.nc` via `arviz.from_netcdf`.

3. Assemble `compare_dict = {display_name: idata for ...}`. Use display names M1..M6b per `BAYESIAN_NETCDF_MAP` convention in `scripts/14_compare_models.py` (for a readable output table).

4. **Pareto-k diagnostic (SOFT GATE — plan-checker Issue #8 Option B):** Before calling `az.compare`, for each model run `loo_result = az.loo(idata, pointwise=True)` and compute `pct_high = float(np.mean(loo_result.pareto_k.values > 0.7) * 100)`. If any model has `pct_high > args.pareto_k_pct_threshold` (default 1.0%), record the value and **log a WARNING but DO NOT exclude the model from `compare_dict`**. Continue the comparison with all convergence-eligible models included. Record `pct_high` per model for the report. The report's "Pareto-k diagnostic" section lists each model's `pct_high` with a clear WARNING marker for any exceeding the threshold — this surfaces the diagnostic for the Wave 4 human checkpoint at the end of this script. (Rationale: Pareto-k exclusion is a scientific judgement call that depends on which model is affected, how large the excess is, and whether alternative methods like K-fold are preferable. Embedding a hard auto-exclusion at this stage would hide the call from the reviewing scientist. The must_haves truth has been updated to match this soft-gate reality.)

5. **Primary — LOO stacking:** `comparison = az.compare(compare_dict, ic="loo", method="stacking")`. Expect an ArviZ 0.23.x DataFrame with columns `rank, elpd_loo, p_loo, elpd_diff, weight, se, dse, warning, scale`.

6. **Weight sum sanity:** `assert abs(comparison["weight"].sum() - 1.0) < 0.01`. If assertion fails, log a WARNING but proceed (ArviZ GitHub #2359 numerical tolerance issue per research doc).

7. **Secondary — RFX-BMS:** For each model, reconstruct per-participant log evidence from `idata.log_likelihood.obs`:
   ```
   # shape (chain, draw, participant, trial_padded)
   # sum over trial axis (NaN-masked padding drops out automatically since filter_padding_from_loglik set them to NaN)
   per_ppt_ll = np.nansum(idata.log_likelihood.obs.values, axis=-1)
   # marginalize chain+draw via logsumexp - log(n_samples)
   log_evidence_per_ppt = scipy.special.logsumexp(per_ppt_ll, axis=(0, 1)) - np.log(per_ppt_ll.shape[0] * per_ppt_ll.shape[1])
   ```
   Assemble `log_evidence_matrix` of shape `(n_participants, n_models)` with consistent participant ordering across models (use sorted participant ids from any of the idata objects — `idata.log_likelihood.participant.values` — and verify all idatas share the same sorted participant list).
   Call `from scripts.fitting.bms import rfx_bms; result = rfx_bms(log_evidence_matrix)`. Unpack `alpha, r, xp, bor, pxp`.

8. **Winner determination:**
   - `w = comparison["weight"]` sorted by rank.
   - If `w.iloc[0] >= 0.5`: winners = [top_model]; winner_type = "DOMINANT_SINGLE".
   - Elif `w.iloc[0] + w.iloc[1] >= 0.8`: winners = [top_model, second_model]; winner_type = "TOP_TWO".
   - Else: winners = all models with `w >= 0.10`; winner_type = "INCONCLUSIVE_MULTIPLE".
   - If `--force-winners` is provided, override with that set.

9. **Write outputs:**
   - `loo_stacking_results.csv`: `comparison.to_csv()` + augmented with the `pct_high_pareto_k` column per model.
   - `rfx_bms_pxp.csv`: columns `model, alpha, r, xp, bor, pxp, pxp_exceeds_95`.
   - `winner_report.md`: sections Summary / Primary (LOO+stacking table) / Secondary (RFX-BMS+PXP table) / Pareto-k diagnostic / Winner verdict / Pipeline action. Include a "USER CHECKPOINT" section at bottom: "Review the winners above; if the automatic determination is appropriate, proceed with plan 21-07. If you want to override, rerun this script with `--force-winners M3,M6b` and then proceed."

10. **Exit codes:**
    - 0: DOMINANT_SINGLE or TOP_TWO, pipeline auto-advances.
    - 2: INCONCLUSIVE_MULTIPLE, pipeline must pause (user reviews winner_report.md and reruns with `--force-winners` OR accepts the multi-winner set).

11. Also write a machine-readable `winners.txt` file containing comma-separated display-name winners (e.g., `M3,M6b`), consumed by plan 21-07 SLURM script.

Follow CLAUDE.md conventions.
