Build step 21.8 — model-averaged scale effects (primary) + optional M6b-subscale exploratory arm.

Purpose: When multiple winner models share the prediction load (Yao et al. 2018 stacking-weighted), aggregate beta-coefficient inferences via a stacking-weighted posterior mixture. This replaces "pick the single-winner HDI" with uncertainty propagation across the model set. If M6b is a winner, also launch the 32-beta subscale exploratory arm per v4.0 Phase 16's existing `wmrl_m6b_hierarchical_model_subscale` (L2-05 requirement).

Output: `output/bayesian/21_l2/averaged_scale_effects.csv` + `averaging_summary.md`. Optional `m6b_subscale_posterior.nc` if M6b is in winners.
</objective>

  <name>Task 1: Implement scripts/21_model_averaging.py with stacking-weighted mixture + optional subscale launcher</name>
  <files>scripts/21_model_averaging.py</files>
  <action>
Create `scripts/21_model_averaging.py`.

**CLI:** `--l2-dir output/bayesian/21_l2/`, `--stacking-results output/bayesian/21_baseline/loo_stacking_results.csv`, `--winners-file output/bayesian/21_baseline/winners.txt`, `--audit-report output/bayesian/21_l2/scale_audit_report.md`, `--output-dir output/bayesian/21_l2/`, `--launch-subscale-arm` (default True if M6b is a winner).

**Workflow:**

1. Read audit report. Parse YAML front-matter header: if `pipeline_action == "NULL_RESULT"`, skip averaging entirely and write `averaging_skipped.md` explaining "No beta sites exclude zero; model averaging would be spurious." Exit 0 (non-fatal).

2. Read `winners.txt` and `loo_stacking_results.csv`. Build `winners_weights = {winner_id: stacking_weight}`. Normalize weights to sum to 1.0 among the winner set.

3. If `len(winners) == 1`: write `single_winner_mode.md` explaining "Only one winner; averaging reduces to single-winner posterior (already in step 21.7 tables). No new averaging artifact produced." Exit 0.

4. For each winner posterior in the winner set:
   - Load `{l2_dir}/{winner}_posterior.nc`.
   - Extract all `beta_*` sites.
   - For each beta site, `.stack(sample=("chain", "draw"))` → 1-D `(n_samples,)` array.
   - Parse each beta site name into a canonical `(covariate_family, target_parameter)` key:
     - Simple 2-cov path sites: `beta_lec_kappa` → `("lec", "kappa")`, `beta_iesr_kappa` → `("iesr", "kappa")`, `beta_lec_kappa_s` → `("lec", "kappa_s")`, etc.
     - M6b subscale sites: `beta_{covariate_name}_{param}` where `covariate_name` ∈ {`lec_total`, `iesr_total`, `iesr_intr_resid`, `iesr_avd_resid`}. Normalize `lec_total` → `lec` and `iesr_total` → `iesr` (stripping the `_total` suffix) so the subscale betas on `lec_total` and `iesr_total` match the simple 2-cov sites by key. Subscale-only families (`iesr_intr_resid`, `iesr_avd_resid`) keep their full name.
   - Subsample `n_sub = int(w * TARGET_TOTAL_SAMPLES)` (where `TARGET_TOTAL_SAMPLES = 8000` and `w = winners_weights[winner]`) without replacement.
   - Group by canonical key across winners: for each `(covariate_family, target_parameter)` key, collect subsamples from every winner that has that key. If a key appears in only ONE winner (e.g., `("iesr_intr_resid", "kappa")` exists only in M6b subscale, never in M3/M5/M6a 2-cov), emit `n_winners_contributing=1`, `single_source=True`, and use the sole winner's samples as-is (no averaging — this is the M6b-only reporting path per user-approved Option C).

5. Compute for each averaged beta (grouped by canonical key):
   - `averaged_mean = np.mean(mixture)`
   - `averaged_sd = np.std(mixture)`
   - `hdi_low, hdi_high = az.hdi(mixture, hdi_prob=0.95)`
   - `excludes_zero = (hdi_low > 0) or (hdi_high < 0)`
   - `tail_p = 2 * min(np.mean(mixture > 0), np.mean(mixture < 0))`
   - If `single_source=True`: copy the single winner's inference; log "[NOTE] {key} reported from {winner} posterior alone; no cross-winner averaging applied (subscale-exclusive family)".

6. Also load the per-winner single-model HDI from `{winner}_beta_hdi_table.csv` (produced by plan 21-08). For each beta site, record `single_winner_excludes_zero` vs `averaged_excludes_zero`, flag disagreements.

7. Write:
   - `averaged_scale_effects.csv`: columns `covariate_family, target_parameter, canonical_key, averaged_mean, averaged_sd, hdi_low, hdi_high, averaged_excludes_zero, tail_p, n_winners_contributing, single_source, source_winner_if_single, disagreement_flag`.
   - `averaging_summary.md`: "Multi-winner averaging engaged. N winners = X. Overlapping keys averaged: K1. Subscale-exclusive keys reported from M6b alone: K2. Top disagreements: ..." — list beta sites where `disagreement_flag = True`.

8. **Optional M6b-subscale exploratory arm:**
   - If `"M6b" in winners` (or `"wmrl_m6b" in winners_internal`) AND `--launch-subscale-arm`:
     - Write a marker file `{output_dir}/launch_subscale.flag` listing the expected SLURM submission.
     - Do NOT sbatch from Python (cluster orchestrator handles this). Instead, the pipeline orchestrator (plan 21-10) reads this flag and conditionally submits `cluster/13_bayesian_m6b_subscale.slurm` with `--output-subdir 21_l2 --export=NAME=m6b_subscale` if the flag exists.
     - Log a loud notice "[SUBSCALE ARM QUEUED] M6b-subscale exploratory arm (32 betas, 12h) queued by orchestrator."

9. Exit 0 on success.

Invocation:
```
python scripts/21_model_averaging.py \
    --l2-dir output/bayesian/21_l2/ \
    --stacking-results output/bayesian/21_baseline/loo_stacking_results.csv \
    --winners-file output/bayesian/21_baseline/winners.txt \
    --audit-report output/bayesian/21_l2/scale_audit_report.md \
    --output-dir output/bayesian/21_l2/
EXIT_CODE=$?

# Handle subscale arm marker
if [ -f output/bayesian/21_l2/launch_subscale.flag ] && [ "${LAUNCH_SUBSCALE:-1}" = "1" ]; then
    echo "[SUBSCALE ARM] M6b is a winner; submitting subscale fit"
    # Subscale arm runs in its own SLURM job; this script doesn't block for it.
    SUBSCALE_JID=$(sbatch --parsable \
        --export=ALL,MODEL=wmrl_m6b,OUTPUT_SUBDIR=21_l2,SUBSCALE_OUTPUT_NAME=m6b_subscale \
        cluster/13_bayesian_m6b_subscale.slurm)
    echo "[SUBSCALE ARM] Submitted as job $SUBSCALE_JID"
    rm output/bayesian/21_l2/launch_subscale.flag
fi

source cluster/autopush.sh
exit $EXIT_CODE
```
