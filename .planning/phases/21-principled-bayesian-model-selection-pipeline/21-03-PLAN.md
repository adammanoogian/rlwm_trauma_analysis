<objective>
Build step 21.2 — Bayesian parameter recovery for all 6 choice-only models using N=50 synthetic datasets per model, fit via hierarchical MCMC.

Purpose: Per Baribault & Collins (2023) gate 3 and Hess et al. (2025) stage 4, demonstrate that identifiable parameters (κ family) recover with Pearson r ≥ 0.80 and HDI coverage ≥ 0.90 under hierarchical inference. Non-identifiable parameters (α±, φ, ρ, K) labeled "descriptive only" per existing MLE recovery findings (quick-005: r = 0.21–0.77).

**Budget decision (resolves research open question #7):** SLURM array job `--array=1-50` with reduced MCMC (warmup=500, samples=1000, chains=2) for each of the 50 synthetic subjects, parallelized across array tasks. At ~1s/iter for fully-batched vmap on CPU, single-subject fit is ~30 min → 50 subjects finish in ~30 min wall clock (array), not the ~21h sequential naive estimate. Per-model total budget: 4 × 30 min = 2h (one model runs 6 times via the master orchestrator).

Output: `output/bayesian/21_recovery/{model}_recovery.csv` with one row per (parameter, subject) pair, plus per-parameter summary rows.

**Scope decision (Phase 21 revision, 2026-04-18):** The 2-covariate L2 hook for M3/M5/M6a (built in plan 21-11) adds two new `beta_lec_*` / `beta_iesr_*` sites per model. Recovery testing for those new β sites is scoped to **plan 21-11's local pytest suite** (`scripts/fitting/tests/test_numpyro_models_2cov.py::test_recovery_2cov_m3`) — NOT to this plan's production 50-subject × 3-model SLURM array. Rationale: adding the L2 hook to the production recovery run would inflate cluster cost by ~3 × 50 = 150 additional MCMC jobs for a confirmatory test that the plan 21-11 pytest already gates at the lighter-weight N=40, short-chain budget. The production recovery in this plan continues to exercise the baseline (no-covariate) inference path for all six choice-only models; the 2-cov path is gated locally before cluster submission via the pytest check in the master orchestrator (plan 21-10).
</objective>

Create `scripts/21_run_bayesian_recovery.py`.

**CLI modes:**
- `--mode single-subject --model <m> --subject-idx <1..50> --seed <int> --output-dir output/bayesian/21_recovery/`
- `--mode aggregate --model <m> --n-subjects 50 --output-dir output/bayesian/21_recovery/`

**single-subject workflow:**
1. Load model params list from `scripts.fitting.numpyro_helpers.PARAM_PRIOR_DEFAULTS` filtered by `MODEL_REGISTRY[model]['params']`.
2. Sample ONE true parameter vector from the prior. For each param: draw `z ~ Normal(0,1)`, then `theta = lower + (upper - lower) * Phi(mu_prior_loc + 0.2 * z)` where `Phi = jax.scipy.stats.norm.cdf`. Use `(subject_idx, seed)` to derive an RNG key so each array task gets different true params.
3. Generate synthetic trial data using `scripts.simulations.generate_data.generate_synthetic_data(model_name, params_dict, n_blocks=17, n_trials_per_block=100)` (or the nearest equivalent — inspect the actual signature and adapt). Save synthetic data DataFrame.
4. Fit the synthetic data via Bayesian MCMC:
   - Build a synthetic `data` DataFrame compatible with `prepare_stacked_participant_data`.
   - Call `scripts.fitting.fit_bayesian._fit_stacked_model(data, model, model_fn, num_warmup=500, num_samples=1000, num_chains=2, seed=<derived>, max_tree_depth=8)`.
   - Extract posterior means + 95% HDI per parameter from the returned MCMC object via `az.summary(idata, var_names=...)` or direct `mcmc.get_samples()`.
5. Compute per-parameter:
   - `posterior_mean` for each param
   - `hdi_low`, `hdi_high` (95%)
   - `in_hdi = int(hdi_low <= true <= hdi_high)` (1/0)
6. Write JSON `output/bayesian/21_recovery/{model}_subject_{idx:03d}.json` with `{true, posterior_mean, hdi_low, hdi_high, in_hdi}` per parameter, plus metadata (MCMC budget, seed, convergence flags — `max_rhat`, `min_ess`, `num_divergences`).

**aggregate workflow:**
1. Glob `output/bayesian/21_recovery/{model}_subject_*.json`.
2. For each parameter, compute:
   - `pearson_r = np.corrcoef(true_values, posterior_means)[0, 1]` (handle NaN if all-identical with warning)
   - `hdi_coverage = np.mean(in_hdi_flags)`
3. Apply the pass criterion:
   - If parameter name in `{"kappa", "kappa_s", "kappa_total", "kappa_share"}`: pass iff `r >= 0.80 AND hdi_coverage >= 0.90`.
   - Otherwise: label `"descriptive only"` — no pass/fail gate.
4. Write `output/bayesian/21_recovery/{model}_recovery.csv` with columns: `model, parameter, n_subjects, pearson_r, hdi_coverage, pass_criterion, status`.
5. Write `output/bayesian/21_recovery/{model}_recovery_summary.md` with a one-paragraph verdict and the table.
6. Exit code 0 if all κ-family params PASS OR if M1/M2 (no κ params). Exit 1 if any κ-family param FAILS.

**Smoke test** `scripts/fitting/tests/test_bayesian_recovery.py`:
- `test_single_subject_recovery_qlearning_smoke()`: run `--mode single-subject --model qlearning --subject-idx 1` with warmup=50, samples=100, chains=1, n_blocks=3, n_trials_per_block=20. - `test_aggregate_handles_missing()`: aggregate with 2 dummy JSONs; verify CSV has correct columns and handles edge cases.

Use: `from __future__ import annotations`, NumPy-style docstrings, absolute imports, Python 3.10+ type hints.
