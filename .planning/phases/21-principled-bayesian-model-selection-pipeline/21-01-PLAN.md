<objective>
Build the step 21.1 prior-predictive-check orchestrator and cluster submission script.

Purpose: Implement the Baribault & Collins (2023, *Psych Methods*, DOI 10.1037/met0000554) prior-predictive gate for all six choice-only models (M1 qlearning, M2 wmrl, M3 wmrl_m3, M5 wmrl_m5, M6a wmrl_m6a, M6b wmrl_m6b). Demonstrates that `PARAM_PRIOR_DEFAULTS` (all `mu_prior_loc=0.0`, v4.0 locked 2026-04-17) produces simulated behavior in the plausible range (median accuracy 0.4–0.9, <10% sub-chance, <5% at-ceiling). Hess et al. (2025, *Comput Psychiatry* 9(1):76–99, DOI 10.5334/cpsy.116) stage-3 equivalent.

Output: `scripts/21_run_prior_predictive.py` + `cluster/21_1_prior_predictive.slurm` + a local smoke test. When run via `sbatch --export=ALL,MODEL=<model>`, writes `output/bayesian/21_prior_predictive/{model}_prior_sim.nc` and a `{model}_gate.md` report.
</objective>

Create `scripts/21_run_prior_predictive.py`. Requirements:

1. Module header: `from __future__ import annotations`, NumPy-style module docstring citing Baribault & Collins (2023) and Hess et al. (2025).
2. CLI (argparse): `--model` (required, one of the 6 choice-only model names), `--data output/task_trials_long.csv`, `--num-draws 500`, `--seed 42`, `--output-dir output/bayesian/21_prior_predictive`.
3. Workflow:
   a. Load data with `scripts.fitting.fit_bayesian.load_and_prepare_data(..., use_cohort=True)` (N=138).
   b. Build stacked arrays via `scripts.fitting.numpyro_models.prepare_stacked_participant_data` + `stack_across_participants`.
   c. Obtain the hierarchical model function from `scripts.fitting.fit_bayesian.STACKED_MODEL_DISPATCH[args.model]`.
   d. Call `numpyro.infer.Predictive(model, num_samples=args.num_draws)(rng_key, **model_kwargs)` with `covariate_lec=None`, `stacked_arrays=<from stack_across_participants>`, `use_pscan=False`. This draws from the prior only (no observations conditioned).
   e. Extract prior-sampled individual-level parameters (e.g., `prior_samples["alpha_pos"]` shape `(num_draws, n_participants)`).
   f. For a random subset of 100 parameter vectors (participant-agnostic: pick one participant index per draw to avoid the full 500 × 138 cartesian), call `scripts.simulations.unified_simulator.simulate_agent_fixed(model_name=args.model, params=<dict>, block_structure=<from data>)` to generate synthetic trial sequences. If `simulate_agent_fixed` has a different signature, inspect it first and adapt — do NOT guess. Falls back to `scripts/simulations/generate_data.py::generate_synthetic_data` if the per-draw simulator path is simpler.
   g. Compute per-draw mean accuracy over all simulated trials; build histogram.
   h. Apply the gate:
      - median(accuracy) in [0.4, 0.9] → PASS else FAIL
      - fraction(draws with accuracy < 0.33) < 0.10 → PASS
      - fraction(draws with accuracy > 0.95) < 0.05 → PASS
   i. Save outputs under `args.output_dir`:
      - NetCDF `{model}_prior_sim.nc` via `arviz.from_dict({"prior": prior_samples, "prior_predictive": {"accuracy": acc_array}})`.
      - CSV `{model}_prior_accuracy.csv` with one row per draw: `draw_idx, accuracy, below_chance, at_ceiling`.
      - Markdown `{model}_gate.md` containing the three gate lines with numeric values and a bold PASS/FAIL verdict.
4. `mkdir -p` the `args.output_dir` using `Path.mkdir(parents=True, exist_ok=True)`.
5. Exit code 0 on PASS, 1 on FAIL so SLURM can surface failures to the dependency chain.

Reference `scripts/fitting/numpyro_helpers.PARAM_PRIOR_DEFAULTS` at import time and print the dict so the log records the exact prior used.

Use absolute imports only. Type hints with Python 3.10+ syntax. Line length 88. NumPy-style docstrings.
