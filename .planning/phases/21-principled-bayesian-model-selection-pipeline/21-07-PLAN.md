<objective>
Build step 21.6 — refit winner model(s) hierarchically WITH L2 scales.

Purpose: Confirmatory scale-effect inference on the winner(s) only. Follows the user-approved Option C: "code for all model variants in principle, but the pipeline will only use one." Plan 21-11 built the 2-covariate L2 infrastructure for M3/M5/M6a (new `covariate_iesr` kwarg, plus `build_level2_design_matrix_2cov` helper). This plan is the runtime dispatcher that reads winners.txt and routes each winner to the correct L2 refit path:

- **M1 (qlearning), M2 (wmrl)** → copy baseline posterior (no L2-compatible parameter target). `shutil.copy2` from 21_baseline/ to 21_l2/, log a warning.
- **M3, M5, M6a** → 2-covariate L2 refit (`lec_total` + `iesr_total`, both z-scored). Uses the new `covariate_iesr` kwarg from plan 21-11 and `build_level2_design_matrix_2cov` to assemble the design. Samples both `beta_lec_{target}` and `beta_iesr_{target}` sites (target = kappa for M3/M5, kappa_s for M6a).
- **M6b** → full 4-covariate subscale design (32 betas = 8 params × 4 covariates) via the existing `wmrl_m6b_hierarchical_model_subscale` path — unchanged from prior revision.

Subscale residual decomposition (iesr_intr_resid + iesr_avd_resid) for M3/M5/M6a is deferred to v5.0 — the current 2-covariate design is the user-approved Option C middle ground.

Output: `output/bayesian/21_l2/{winner}_posterior.nc` etc., for each winner in `winners.txt`.

**Runtime estimate (updated for 2-cov L2):** The 2-covariate path for M3/M5/M6a adds one additional numpyro sample site per target parameter (2 total betas vs 1 in Phase 16). Posterior geometry is near-identical — the additional dimension is decoupled by construction (z-scored covariates, same Normal(0, 1) prior). Estimated wall-clock per L2-capable winner at production budget (chains=4, warmup=1000, samples=2000, max_tree_depth=10): M3/M5/M6a ~45–75 min (was ~30–45 min single-predictor, ~1.5× due to extra beta site + any tree-depth increase); M6b subscale ~8–12 h (unchanged). 12h SLURM time cap absorbs M6b worst case; 8h cap adequate for the 2-cov path.
</objective>
