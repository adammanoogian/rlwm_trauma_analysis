  Note the numbering gap (M3 -> M5) is intentional; M4 is listed last because it is the only joint choice+RT model. Add a note: "M4 uses joint choice+RT likelihood; its AIC is not comparable to choice-only models M1-M3, M5, M6."

  **New Section 3.6 (after existing Section 3.5):** M3 WM-RL + Perseveration (kappa)
  - Extends M2 with a choice kernel C_k for the last action taken
  - On each trial, before softmax: hybrid_with_persist(a|s) = (1-kappa) * hybrid(a|s) + kappa * C_k(a)
  - C_k(a) = 1 if a was the last action for stimulus s, 0 otherwise (one-hot)
  - At first presentation of a stimulus in a block, no kernel applied (uniform fallback)
  - Parameter table: 7 params matching WMRL_M3_PARAMS in mle_utils.py
  - kappa in [0,1]: strength of perseveration; kappa=0 reduces to M2

  **New Section 3.7:** M5 WM-RL + RL Forgetting (phi_rl)
  - Extends M3 with Q-value decay before delta-rule update
  - Each trial, BEFORE the standard delta-rule: Q(s,a) <- (1-phi_rl)*Q(s,a) + phi_rl*Q0 for ALL (s,a) pairs
  - Q0 = 1/nA = 0.333 (uniform baseline, matching WM baseline)
  - Then standard asymmetric delta-rule update for the observed (s,a) pair
  - Parameter table: 8 params matching WMRL_M5_PARAMS
  - phi_rl in [0,1]: decay rate; phi_rl=0 reduces exactly to M3

  **New Section 3.8:** M6a Stimulus-Specific Perseveration (kappa_s)
  - Replaces M3's global kappa with per-stimulus tracking
  - last_actions array (shape num_stimuli), initialized to -1 (sentinel for "not yet seen")
  - On each trial for stimulus s: if last_actions[s] >= 0, apply kernel with kappa_s; else uniform fallback
  - After action, unconditionally update: last_actions[s] = action_taken
  - Reset all last_actions to -1 at block boundaries
  - Parameter table: 7 params matching WMRL_M6A_PARAMS
  - Note: M6a has same number of free parameters as M3 but different mechanism

  **New Section 3.9:** M6b Dual Perseveration (stick-breaking)
  - Combines global (M3-style) and stimulus-specific (M6a-style) perseveration
  - Stick-breaking reparameterization: kappa_total in [0,1], kappa_share in [0,1]
  - Decoded: kappa = kappa_total * kappa_share; kappa_s = kappa_total * (1 - kappa_share)
  - Enforces kappa + kappa_s = kappa_total <= 1
  - Dual carry: both global last_action (scalar) and per-stimulus last_actions (array)
  - Effective-weight gating: if stimulus not yet seen in block, kappa_s portion = 0
  - kappa_share=1 reduces to M3; kappa_share=0 reduces to M6a
  - Parameter table: 8 params matching WMRL_M6B_PARAMS

  **New Section 3.10:** M4 RLWM-LBA Joint Choice+RT
  - Learning: Same as M3 (hybrid policy with kappa perseveration)
  - Decision: Linear Ballistic Accumulator (Brown & Heathcote, 2008)
  - Drift rates: v_i = v_scale * pi_hybrid(a_i | s_t) for each accumulator
  - Start points: k_i ~ Uniform(0, A)
  - Threshold: b = A + delta (b > A by construction via reparameterization)
  - Within-trial noise: s = 0.1 (fixed, not a free parameter)
  - RT = accumulation_time(winner) + t0 (non-decision time)
  - Joint log-likelihood: log P(choice=i, RT=t) = log f_i(t) + sum_{j!=i} log S_j(t)
  - f_i(t) = LBA PDF for accumulator i; S_j(t) = LBA survivor function for accumulator j
  - No epsilon parameter (start-point variability A subsumes undirected exploration)
  - Requires float64 for numerical stability (CDF computations)
  - Parameter table: 10 params matching WMRL_M4_PARAMS
  - AIC NOT comparable to choice-only models (different likelihood domains)

  **Section 4.2 revision:** Remove "information sharing (i)" from the "Intentional Simplifications" list. Replace with a note that perseveration has been implemented as M3's kappa parameter and extended in M5, M6a, M6b, M4.

  **Section 9 (Code Structure) update:** Add `lba_likelihood.py` to the fitting module file listing with description "LBA density functions for M4 joint choice+RT (float64)".
