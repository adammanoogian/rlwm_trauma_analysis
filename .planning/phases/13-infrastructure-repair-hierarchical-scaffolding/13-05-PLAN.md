    **Functions to implement:**

    1. `compute_pointwise_log_lik(mcmc, participant_data: dict, model_name: str, *, num_stimuli: int = 6, num_actions: int = 3, q_init: float = 0.5) -> jnp.ndarray`:
       - Takes an MCMC object (after sampling) and participant data dict.
       - Returns array of shape `(chains, samples, n_participants, max_total_trials)`.
       - Implementation approach:
         a. Extract posterior samples grouped by chain: `samples = mcmc.get_samples(group_by_chain=True)`.
         b. For each participant, construct the stacked arrays needed by the `*_multiblock_likelihood_stacked` function.
         c. Use the appropriate likelihood function based on `model_name` (dispatch on MODEL_REGISTRY or match "qlearning" / "wmrl" / "wmrl_m3" / etc.).
         d. Call the stacked function with `return_pointwise=True` for each (chain, sample, participant) combination.
         e. Use `jax.vmap` to vectorize over samples within a chain, then loop over chains (or vmap over chains too if memory allows).
       - The research doc sketch (line 194-223) shows the pattern. Adapt it to use `*_multiblock_likelihood_stacked` instead of `*_multiblock_likelihood`.
       - Parameter extraction from samples: each model has different parameters. Use `model_name` to determine which sample keys to extract (e.g., "qlearning" needs alpha_pos, alpha_neg, epsilon; "wmrl_m3" also needs phi, rho, capacity, kappa).
       - IMPORTANT: The `samples[param_name]` array has shape `(chains, samples_per_chain, n_participants)` for individual-level params. Index into the participant dimension.

    2. `build_inference_data_with_loglik(mcmc, pointwise_log_lik: jnp.ndarray, *, participant_ids: list | None = None) -> "az.InferenceData"`:
       - Converts MCMC to ArviZ InferenceData and adds the log_likelihood group.
       - Implementation:
         ```python
         import arviz as az
         idata = az.from_numpyro(mcmc)
         n_ppts = pointwise_log_lik.shape[2]
         n_trials = pointwise_log_lik.shape[3]
         idata.add_groups(
             log_likelihood={"obs": pointwise_log_lik},
             dims={"obs": ["participant", "trial"]},
         )
         return idata
         ```
       - After this, `az.waic(idata)` and `az.loo(idata)` work natively.

    3. `_get_likelihood_fn(model_name: str)`:
       - Internal dispatch function. Returns the appropriate `*_multiblock_likelihood_stacked` function for the given model name.
       - Map: "qlearning" -> `q_learning_multiblock_likelihood_stacked`, "wmrl" -> `wmrl_multiblock_likelihood_stacked`, "wmrl_m3" -> `wmrl_m3_multiblock_likelihood_stacked`, etc.
       - Import the functions from `scripts.fitting.jax_likelihoods`.

    4. `_get_param_names(model_name: str) -> list[str]`:
       - Returns the list of parameter names for a model.
       - Map: "qlearning" -> ["alpha_pos", "alpha_neg", "epsilon"], "wmrl_m3" -> ["alpha_pos", "alpha_neg", "phi", "rho", "capacity", "kappa", "epsilon"], etc.
       - Can import from MODEL_REGISTRY in config.py if it has this info, or hardcode.

    **Design notes:**
    - Do NOT import from `numpyro_helpers.py` — this module is independent (diagnostics only, no prior construction).
    - The pointwise log-lik computation is the most compute-intensive part. Use `jax.jit` on the per-participant vmap'd function.
    - Padding masks: the `*_multiblock_likelihood_stacked` functions already return 0.0 for padded trials. WAIC/LOO computations naturally handle zero-contribution trials.
    - Cite Senta 2025 p.22 in the module docstring for BIC rejection rationale.

    **Part A: Create `scripts/fitting/bayesian_summary_writer.py`.**

    ```python
    """Schema-parity CSV writer for Bayesian hierarchical fits.

    Converts ArviZ InferenceData to CSV format matching MLE output schema,
    enabling downstream scripts (15, 16, 17) to consume via --source bayesian
    with zero logic changes.

    BIC is reported for v3.0 back-compatibility, but WAIC/LOO is the primary
    Bayesian criterion (Senta, Bishop, Collins 2025, p.22).

    v4.0 INFRA-04.
    """
    from __future__ import annotations
    ```

    **Functions:**

    1. `write_bayesian_summary(idata: "az.InferenceData", model_name: str, output_dir: Path, *, param_names: list[str], participant_ids: list, parameterization_version: str) -> Path`:
       - Converts InferenceData posterior to a DataFrame matching MLE schema.
       - For each participant, each parameter:
         - `{param}` column: posterior MEAN (matches MLE point estimate semantics)
         - `{param}_hdi_low` column: 95% HDI lower bound
         - `{param}_hdi_high` column: 95% HDI upper bound
         - `{param}_sd` column: posterior standard deviation
       - Outcome columns:
         - `nll`: negative log-likelihood at posterior mean (compute by evaluating likelihood at posterior mean params)
         - `aic`: 2*k + 2*nll (for back-compat; NOT primary criterion)
         - `bic`: k*ln(n) + 2*nll (for back-compat; NOT primary criterion)
         - `aicc`: aic + 2*k*(k+1)/(n-k-1) (for back-compat)
         - `pseudo_r2`: 1 - nll/null_nll (for back-compat)
       - Convergence columns:
         - `max_rhat`: max R-hat across all parameters for this participant (from `az.summary`)
         - `min_ess_bulk`: min ESS bulk across all parameters
         - `num_divergences`: from idata.sample_stats
       - Standard columns:
         - `n_trials`: number of real trials (non-padding)
         - `converged`: True if max_rhat < 1.01 AND min_ess_bulk > 400 AND num_divergences == 0
         - `at_bounds`: empty string (not applicable to Bayesian)
       - Metadata:
         - `parameterization_version`: the version string
       - Columns that are in MLE but NOT in Bayesian (dropped): `grad_norm`, `hessian_condition`, `hessian_invertible`, `*_se`, `*_ci_lower`, `*_ci_upper`, `n_successful_starts`, `n_near_best`, `high_correlations`.
       - Output path: `{output_dir}/bayesian/{model_name}_individual_fits.csv`
       - Create output directory if needed.

    2. `_compute_participant_nll(idata, participant_idx: int, model_name: str, participant_data: dict) -> float`:
       - Helper to compute NLL at posterior mean for AIC/BIC back-compat.
       - Extract posterior means, call likelihood function, return -log_lik.
