4. **generate_synthetic_participant()**: This is the MAJOR change. Add M4 support with RT simulation.

   a. Parameter extraction: When `model == 'wmrl_m4'`:
      - Extract same learning params as M3: alpha_pos, alpha_neg, phi, rho, capacity, kappa
      - Extract LBA params: v_scale, A, delta, t0
      - Compute: `b = A + delta` (reparameterization decode)
      - Set `epsilon = 0.0` for the noise step but SKIP epsilon application entirely (set a flag `use_epsilon = (model != 'wmrl_m4')`)
      - Also extract phi_rl = 0.0 (no RL forgetting in M4), kappa_s = 0.0 (no stim-specific kernel)

   b. Add M4 to model lists: `model in ['wmrl', 'wmrl_m3', 'wmrl_m5', 'wmrl_m6a', 'wmrl_m6b', 'wmrl_m4']` for WM-RL paths.

   c. **Epsilon application**: Currently epsilon is applied unconditionally:
      ```python
      noisy_probs = epsilon / NUM_ACTIONS + (1 - epsilon) * action_probs
      ```
      For M4, SKIP this step. The simplest approach:
      ```python
      if model == 'wmrl_m4':
          noisy_probs = action_probs  # No epsilon; A subsumes exploration
      else:
          noisy_probs = epsilon / NUM_ACTIONS + (1 - epsilon) * action_probs
      ```

   d. **Action selection**: For M4, instead of sampling from noisy_probs, simulate the LBA race:
      ```python
      if model == 'wmrl_m4':
          # LBA race simulation (McDougle & Collins 2021)
          v_all = v_scale * action_probs  # drift rates from hybrid policy
          # Start points: k_i ~ Uniform(0, A)
          k = rng.uniform(0, A, size=NUM_ACTIONS)
          # Time to threshold: t_i = (b - k_i) / v_i
          # Handle zero/negative drift: use very large time
          v_safe = np.maximum(v_all, 1e-6)
          t_race = (b - k) / v_safe
          # Winner is the accumulator that reaches threshold first
          winner = int(np.argmin(t_race))
          rt_sim = float(t_race[winner] + t0)  # RT = finishing time + non-decision time
          action = winner
      else:
          # Standard probabilistic choice
          key, subkey = jax.random.split(key)
          action = int(jax.random.choice(subkey, NUM_ACTIONS, p=jnp.array(noisy_probs)))
          rt_sim = None  # Not used for choice-only models
      ```

   e. **Trial data**: Add RT to the trial dict when model is wmrl_m4:
      ```python
      trial_dict = {
          'sona_id': sona_id,
          'block': block_num,
          'stimulus': stimulus,
          'key_press': action,
          'reward': reward,
          'set_size': set_size,
          'trial_in_block': trial_in_block

      if model == 'wmrl_m4':

          trial_dict['rt'] = rt_sim * 1000.0
      all_trials.append(trial_dict)
      ```

   f. **Perseveration**: M4 uses global kappa (same as M3). CRITICAL: The synthetic generator MUST use the SAME convex combination formula as the M4 likelihood (Plan 02, Task 1, step f). Do NOT use additive renormalization. Add 'wmrl_m4' to the perseveration condition, but use the convex combination form for M4:
      ```python
      if model in ('wmrl_m3', 'wmrl_m5', 'wmrl_m4') and last_action is not None:
          # Convex combination (matches likelihood formula exactly)
          Ck = np.zeros(NUM_ACTIONS)
          Ck[last_action] = 1.0
          hybrid_probs = (1.0 - kappa) * hybrid_probs + kappa * Ck
      ```
      NOTE: For M3 and M5, the existing additive renormalization formula is used by those models' likelihoods, so ONLY change the M4 path. The cleanest approach:
      ```python
      if last_action is not None:
          if model == 'wmrl_m4':
              # Convex combination (must match M4 likelihood exactly)
              Ck = np.zeros(NUM_ACTIONS)
              Ck[last_action] = 1.0
              hybrid_probs = (1.0 - kappa) * hybrid_probs + kappa * Ck
          elif model in ('wmrl_m3', 'wmrl_m5'):
              # Additive renormalization (matches M3/M5 likelihood)
              hybrid_probs = hybrid_probs.copy()
              hybrid_probs[last_action] += kappa
              hybrid_probs = hybrid_probs / np.sum(hybrid_probs)
      ```
      This ensures the synthetic data generation matches each model's likelihood formula exactly, preventing kappa recovery bias.
