<objective>
Implement the M6b dual-perseveration model: JAX likelihood functions with BOTH global last_action scalar (M3-style) AND per-stimulus last_actions array (M6a-style) in the lax.scan carry, stick-breaking reparameterization (kappa_total, kappa_share) for constraint enforcement, and full MLE fitting pipeline integration.

Purpose: M6b tests whether perseveration has both a global motor component (repeat last action regardless of stimulus) and a stimulus-specific component (repeat last action for THIS stimulus). The stick-breaking parameterization ensures kappa + kappa_s <= 1 by construction, creating a smooth optimization landscape. M6b has 8 free parameters: alpha_pos, alpha_neg, phi, rho, capacity, kappa_total, kappa_share, epsilon.

Output: Three files modified such that `python scripts/12_fit_mle.py --model wmrl_m6b` runs end-to-end with dual perseveration kernels and stick-breaking constraint.
</objective>

**In jax_likelihoods.py**, add M6b likelihood functions. M6b combines M3's global kernel and M6a's per-stimulus kernel with a dual carry:

1. Add `wmrl_m6b_block_likelihood(stimuli, actions, rewards, set_sizes, alpha_pos, alpha_neg, phi, rho, capacity, kappa, kappa_s, epsilon, num_stimuli, num_actions, q_init, wm_init, mask)` after the M6a block likelihood.
   CRITICAL: The function signature takes DECODED `kappa` and `kappa_s` (not kappa_total/kappa_share). The stick-breaking decode happens in the objective functions that call this.

   **DUAL CARRY**: Initialize BOTH tracking structures:
   ```python
   last_action_init = -1  # scalar int, same as M3
   last_actions_init = jnp.full((num_stimuli,), -1, dtype=jnp.int32)
   init_carry = (Q_init, WM_init, WM_0, 0.0, last_action_init, last_actions_init)
   ```

   Inside the step_fn, unpack the 6-element carry:
   `Q_table, WM_table, WM_baseline, log_lik_accum, last_action, last_actions = carry`

   The step_fn body follows this order:
   a. WM decay: `WM_decayed = (1 - phi) * WM_table + phi * WM_baseline`
   b. Compute hybrid policy (Q + WM with omega weighting), apply softmax, apply epsilon noise to get `P_noisy`
   c. Global kernel (M3 component):
      ```python
      has_global = jnp.logical_and(kappa > 0.0, last_action >= 0)
      Ck_global = jnp.eye(num_actions)[jnp.maximum(last_action, 0)]
      eff_kappa = jnp.where(has_global, kappa, 0.0)
      ```
   d. Stimulus-specific kernel (M6a component):
      ```python
      last_action_s = last_actions[stimulus]
      has_stim = jnp.logical_and(kappa_s > 0.0, last_action_s >= 0)
      Ck_stim = jnp.eye(num_actions)[jnp.maximum(last_action_s, 0)]
      eff_kappa_s = jnp.where(has_stim, kappa_s, 0.0)
      ```
   e. Three-way blend:
      ```python
      noisy_probs = (
          (1.0 - eff_kappa - eff_kappa_s) * P_noisy
          + eff_kappa * Ck_global
          + eff_kappa_s * Ck_stim
      )
      ```
      NOTE: No `use_m2_path` branch needed -- the effective weights handle all four gating cases.
   f. Log-likelihood: `log_prob = jnp.log(noisy_probs[action] + 1e-8)` masked by valid.
   g. Update Q and WM (identical to M3/M6a).
   h. Update BOTH perseveration states:
      ```python
      new_last_action = jnp.where(valid, action, last_action).astype(jnp.int32)
      new_last_actions = last_actions.at[stimulus].set(
          jnp.where(valid, action, last_action_s).astype(jnp.int32)
      )
      ```
   i. Return 6-element carry: `(Q_updated, WM_updated, WM_baseline, log_lik_new, new_last_action, new_last_actions), log_prob_masked`
