```python

    if args.model != 'wmrl_m3':
```

3. **Modified fit path** when `--permutation-shuffle` is active:

Instead of calling `fit_model()` directly, the permutation path must:
a. Prepare stacked data and load LEC covariate (same as _fit_stacked_model)
b. Shuffle the LEC covariate: `rng = np.random.default_rng(args.permutation_shuffle); covariate_lec = jnp.array(rng.permutation(np.array(covariate_lec)))`
c. Use reduced MCMC budget: `num_warmup=500, num_samples=1000` (half of standard, sufficient for HDI-excludes-zero check)
d. Run inference with the shuffled covariate
e. Extract `beta_lec_kappa` from samples, compute 95% HDI
f. Save results to `output/bayesian/permutation/shuffle_{idx}_results.json`:
```json
{
    "shuffle_idx": 0,
    "beta_lec_kappa_mean": 0.012,
    "beta_lec_kappa_std": 0.045,
    "hdi_low": -0.078,
    "hdi_high": 0.102,
    "excludes_zero": false,
    "n_divergences": 0,
    "wall_clock_seconds": 3600
}
```
g. Skip `save_results()` (no need for full CSV/NetCDF/shrinkage for permutation runs)

IMPORTANT: The permutation shuffles the LEC covariate ROWS (participant-level labels), not within-participant data. The covariate array is aligned to `sorted(participant_data_stacked.keys())`, and the permutation randomizes this alignment.

**SLURM array job** (`cluster/13_bayesian_permutation.slurm`):

Copy `cluster/13_bayesian_m3.slurm` as base. Modifications:
```bash
#SBATCH --job-name=perm_null
#SBATCH --output=logs/perm_null_%A_%a.out
#SBATCH --error=logs/perm_null_%A_%a.err
#SBATCH --array=0-49
#SBATCH --time=04:00:00
#SBATCH --mem=32G
#SBATCH --cpus-per-task=4
#SBATCH --partition=comp

# ... same conda/JAX setup ...

python scripts/fitting/fit_bayesian.py \
    --model wmrl_m3 \
    --data output/task_trials_long.csv \
    --permutation-shuffle $SLURM_ARRAY_TASK_ID \
    --chains 4 \
    --warmup 500 \
    --samples 1000 \
    --seed 42
```

Note: `%A` is the master job ID, `%a` is the array task ID in SLURM log filenames.
Time reduced to 4h (half budget runs faster). The --warmup and --samples override the defaults.

**Aggregation script** (`scripts/fitting/aggregate_permutation_results.py`):

Short script that:
1. Globs `output/bayesian/permutation/shuffle_*_results.json`
2. Loads each JSON
3. Counts how many have `"excludes_zero": true`
4. Reports: `{n_surviving}/{n_total} shuffles had HDI excluding zero ({pct}%)`
5. Writes summary to `output/bayesian/permutation/permutation_summary.md`:
```
# Permutation Null Test (L2-06)

## Results
- Shuffles completed: {n}
- Surviving effects (HDI excludes zero): {n_surviving}
- False positive rate: {pct}%
- Nominal alpha: 5%

## Verdict
{PASS if n_surviving/n_total <= 0.05 else FAIL}
```

Use `from __future__ import annotations`, `argparse` for `--output-dir` flag, NumPy-style docstring.
