**M6b (HIER-06):** This is the most complex model. Key differences from M3:

1. **8 parameters**: alpha_pos, alpha_neg, phi, rho, capacity, kappa_total, kappa_share, epsilon.
2. **Two manually-sampled params** (kappa_total AND kappa_share) — both need the manual sampling pattern with optional L2 shift, NOT `sample_bounded_param`.
3. **Standard params via sample_bounded_param**: alpha_pos, alpha_neg, phi, rho, capacity, epsilon (6 params).
4. **kappa_total** sampling: Same pattern as M3's kappa — `kappa_total_mu_pr`, `kappa_total_sigma_pr`, `kappa_total_z`, with optional `beta_lec_kappa_total` L2 shift.
5. **kappa_share** sampling: Same pattern — `kappa_share_mu_pr`, `kappa_share_sigma_pr`, `kappa_share_z`, with optional `beta_lec_kappa_share` L2 shift.
6. **Decode per-participant** (CRITICAL, locked in STATE.md):
   ```python
   for idx, pid in enumerate(participant_ids):
       kappa_total_i = sampled["kappa_total"][idx]
       kappa_share_i = sampled["kappa_share"][idx]
       kappa   = kappa_total_i * kappa_share_i
       kappa_s = kappa_total_i * (1.0 - kappa_share_i)
       log_lik = wmrl_m6b_multiblock_likelihood_stacked(
           ..., kappa=kappa, kappa_s=kappa_s, ...
       )
       numpyro.factor(f"obs_p{pid}", log_lik)
   ```
7. The likelihood takes decoded `kappa` and `kappa_s`, NOT `kappa_total`/`kappa_share`.

The L2 coefficient pattern: If `covariate_lec` is not None, sample TWO beta coefficients:
```python
beta_lec_kappa_total = numpyro.sample("beta_lec_kappa_total", dist.Normal(0.0, 1.0))
```

Each shifts the corresponding unconstrained parameter:

IMPORTANT: The `PARAM_PRIOR_DEFAULTS` for `kappa_total` has `mu_prior_loc=-2.0` and `kappa_share` has `mu_prior_loc=0.0`. Use these as the locations for the respective `mu_pr` priors.
